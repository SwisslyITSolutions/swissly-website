# English i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the German Astro site fully bilingual (DE default at `/`, EN at `/en/`) without changing any German page.

**Architecture:** Astro native i18n routing (`prefixDefaultLocale: false`). German content/data stays untouched; English lives in parallel `data/en/*.ts` modules and `src/pages/en/*` pages. In-component UI strings move to a central `src/i18n/ui.ts` dictionary read via a `t(lang, key)` helper. Components derive language from `Astro.currentLocale` (fallback `'de'`). Layout/SEO become language-aware (`<html lang>`, `og:locale`, `hreflang` alternates).

**Tech Stack:** Astro 6.1 (static output), TypeScript, Tailwind v4, Vitest (140 unit tests), Playwright (e2e).

## Global Constraints

- **German pages must not change** — content, markup, or visual output. DE data modules (`src/data/*.ts`) and existing tests stay byte-identical unless a task explicitly says otherwise.
- **English variant:** British/international spelling (organisation, optimise, centre, programme).
- **Swiss facts preserved verbatim** in EN: CHF prices (CHF 140/hour, from CHF 59/mo), TWINT/Stripe/Datatrans/Wallee, Ebikon/Lucerne/Central Switzerland, 15-minute billing, Lighthouse 90+, Foundation/Growth/Scale, "301 redirects".
- **No em-dash `—` in body copy** (DE or EN) — use period/comma/colon. `|` is the title separator.
- **Legal pages stay German only** (AGB, Datenschutz, Impressum). EN footer links point to the DE legal pages, labelled "(German)".
- `trailingSlash: 'always'` and `output: 'static'` remain.
- After every task: `npm run build` clean AND `npm test` 140/140 green.

## File Structure

**New files:**
- `src/i18n/ui.ts` — UI-string dictionary `{ de: {...}, en: {...} }` + `Lang` type + `t(lang, key)` helper.
- `src/i18n/utils.ts` — `getLang(Astro)` (reads `Astro.currentLocale`, fallback `'de'`), `localizedPath(lang, dePath)` (maps a DE path to its EN counterpart for the switcher).
- `src/i18n/content.ts` — language accessor over data modules: `getContent(lang)` returning `{ services, faq, team, careplans, pricing, process, values, ecs }`.
- `src/data/en/{services,faq,team,careplans,pricing,process}.ts` — EN content, same shape/types as DE.
- `src/components/layout/LanguageSwitcher.astro` — `DE | EN` control.
- `src/pages/en/{index,erstgespraech,faq,ueber-uns,404}.astro`, `src/pages/en/leistungen/{index,[slug]}.astro`, `src/pages/en/kontakt/{index,danke}.astro` — EN pages.

**Modified files:**
- `astro.config.mjs` — add `i18n` block.
- `src/components/seo/Seo.astro` — add `lang` + `alternates` props; emit `og:locale` and `hreflang`.
- `src/layouts/BaseLayout.astro` — language-aware `<html lang>`, pass `lang`/alternates to `Seo`.
- `src/layouts/ServiceLayout.astro` — language-aware.
- `src/components/sections/*.astro`, `src/components/layout/{Header,Footer,MobileMenu,StatusBar}.astro`, relevant `src/components/ui/*` — replace hardcoded UI strings with `t(getLang(Astro), '…')`.
- `src/data/site.ts` — split language-dependent strings (claim, tagline, values, nav labels, ctaLabel, statusBar, ecs.text) into the i18n layer; keep neutral constants (email, address, hours, hourlyRate, social, commercialRegister).

- [ ] **Step 4: Implement `src/i18n/utils.ts`:**

```ts
export type Lang = 'de' | 'en';

export function getLang(astro: { currentLocale?: string }): Lang {
  return astro.currentLocale === 'en' ? 'en' : 'de';
}

/** Map a path between DE (root) and EN (/en/ prefix). Accepts either-language input. */
export function localizedPath(lang: Lang, path: string): string {
  const bare = path.replace(/^\/en(?=\/|$)/, '') || '/';
  if (lang === 'en') return bare === '/' ? '/en/' : `/en${bare}`;
  return bare;
}
```

- [ ] **Step 5: Run** `npx vitest run tests/i18n-utils.test.ts` → PASS. Then `npm run build` → clean (DE pages still at root).
