# Redirect Map — swisslyit.ch

## Overview

The Swissly IT Solutions website uses Astro with `trailingSlash: 'always'` and is
deployed to Hostpoint shared hosting (Apache 2.4). All URL routing is handled by
`public/.htaccess`. No slug changes were made during the site build, so only
infrastructure-level redirects are required.

---

## 1. Active Redirect Rules

| Rule | From | To | Status |
|------|------|----|--------|
| Force HTTPS | `http://swisslyit.ch/*` | `https://swisslyit.ch/*` | 301 |
| www → apex | `https://www.swisslyit.ch/*` | `https://swisslyit.ch/*` | 301 |
| No trailing slash → trailing slash | `https://swisslyit.ch/leistungen` | `https://swisslyit.ch/leistungen/` | 301 |

The trailing-slash rule applies to all directory-style paths that do not end in
`/` and do not have a file extension. It is driven by Astro's
`trailingSlash: 'always'` setting and the `.htaccess` `RewriteRule` in section 1c.

## 3. What is NOT Needed

- **No slug changes**: The URL structure from the previous site is preserved
  exactly. There are no old slugs that need redirecting to new ones.
- **No hash/query-string redirects**: No legacy `?page=` or `#anchor` patterns
  were in use that require rewriting.
- **No subdomain redirects beyond www**: Only `www` → apex is handled; no other
  subdomains exist.
