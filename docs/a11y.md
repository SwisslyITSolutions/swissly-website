# Accessibility Audit — Swissly IT Solutions Website

## Static Analysis (dist HTML)

Audit performed against `dist/` build output. No browser required.

---

### HTML Language

- `<html lang="de-CH">` present on all pages.

---

### Heading Structure

- **Single `<h1>` per page** — confirmed on all audited pages:
  - `/` → "Webseiten, die einfach laufen."
  - `/leistungen/` → "Alles aus einer Hand."
  - `/kontakt/` → "Sprechen wir miteinander."
  - `/faq/` → "Häufige Fragen"
- Footer section headings use `<h2>` (Navigation, Kontakt) — appropriate level.
- Service cards use `<h3>` — correct hierarchy.

---

### Images & Alt Text

- **Logo mark** (`logo-mark.svg`) in header: `alt=""` — correct, it is decorative (the adjacent text "SWISSLY IT" provides the label via `aria-label` on the parent `<a>`).
- **Logo banner** in footer: `alt="Swissly IT Solutions"` — descriptive alt text present.
- All decorative background divs, glows, and SVG icons carry `aria-hidden="true"`.
- No raster images are rendered on any page. `public/images/og-default.png` exists solely as the Open Graph share image (referenced in `<meta property="og:image">`), not rendered in page content.

---

### Skip Link

- "Zum Inhalt springen" skip link present on all pages, linking to `#main`.
- Visible on focus (`focus:not-sr-only`) with gold background — meets WCAG 2.4.1.

---

### Form Labels (Kontakt page)

All form fields have explicit `<label for="...">` associations:

| Field | Label id | Input id | Status |
|---|---|---|---|
| Name / Firma | — | `field-name` | `<label for="field-name">` |
| E-Mail | — | `field-email` | `<label for="field-email">` |
| Telefon (optional) | — | `field-telefon` | `<label for="field-telefon">` |
| Ihr Anliegen | — | `field-anliegen` | `<label for="field-anliegen">` |
| Datenschutz consent | — | — | Wrapping `<label>` with text |

Honeypot field: `aria-hidden="true"`, `tabindex="-1"` — correctly hidden from AT.

---

### Navigation / ARIA

- **Desktop nav**: `<nav aria-label="Hauptnavigation">` — named landmark.
- **Mobile menu**: `role="dialog"`, `aria-modal="true"`, `aria-label="Navigation"`.
  - Full keyboard trap (Tab/Shift+Tab cycles within modal).
  - Escape key closes and returns focus to toggle button.
  - Toggle button: `aria-expanded`, `aria-controls="mobile-menu"`, `aria-label="Menü öffnen/schliessen"`.
- **Footer nav**: `<nav aria-label="Footer-Navigation">` — named landmark.

---

### FAQ Accordion

Each accordion item uses correct ARIA pattern:
- Trigger: `<button aria-expanded="false/true" aria-controls="acc-*-panel">` inside `<h3>`.
- Panel: `role="region"`, `aria-labelledby="acc-*-btn"`.
- Icon `+/×`: `aria-hidden="true"`.

---

### Cookie Banner

- `role="region"`, `aria-label="Cookie-Hinweis"`, `aria-live="polite"`.
- Two action buttons: "Akzeptieren" and "Ablehnen" — keyboard accessible.
- "Einstellungen" link provides additional control.
- Footer "Cookie-Einstellungen" button re-opens banner via `window.swisslyConsent.open()`.

---

### Map (Kontakt page)

- Leaflet map container: `role="application"`, `aria-label="Karte: Swissly IT Solutions KLG, Pilatusweg 23, 6030 Ebikon"`.
- Loaded lazily via IntersectionObserver — no CLS or blocking load.
- Static fallback text "Karte wird geladen…" shown before JS runs.

---

### Potential Minor Findings

| Finding | Severity | Notes |
|---|---|---|
| Cookie banner uses `role="region"` not `role="dialog"` | Informational | Region is acceptable for a banner; dialog would require focus management. Current approach (persistent bottom banner) suits region semantics. |
| Leaflet map focus management | Low | After lazy-load, Leaflet map becomes keyboard-navigable via arrow keys. `role="application"` correctly signals this to screen readers. |
| Consent checkbox has no `id` | Informational | Uses wrapping `<label>` pattern — WCAG-compliant alternative to `for`/`id`. |

---

## Automated Test Coverage

See `tests/e2e/a11y.spec.ts` for Playwright + axe-core tests covering:
- WCAG 2.0 A/AA and 2.1 AA rules on all four main pages
- Keyboard navigation (desktop nav, FAQ accordion)
- Cookie banner interaction

### Selector notes

- Accordion triggers use class `.accordion-trigger` (not `[data-accordion-trigger]`).
- Cookie banner uses `role="region"` with `aria-label="Cookie-Hinweis"` (not `role="dialog"`).
  Target via `[role="region"][aria-label*="Cookie"]` — do not use `[role="dialog"][aria-label]`
  which would match the mobile menu instead.

### Status

Automated E2E tests MUST be run at go-live: `npx playwright install chromium` was not
feasible in the build environment. Run after deploy:

```bash
npm run build
npm run preview &
npx playwright test tests/e2e/a11y.spec.ts
```

Or use the axe DevTools browser extension on the live site.
